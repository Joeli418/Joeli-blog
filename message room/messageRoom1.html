Here are the things you want to do.

try connecting the multiplexer first with the FIRST 3 BIT of the DIP switch.
The DIP switch is configured as following
  • bit 1-3: instruction bit
  • bit 5-7: designation register
  • bit 9-11: input register 1
  • bit 13-15: input register 2
the rest of the bit should serve no purpose

try making the MOVE instruction first with 000. Make sure it works. The computer should work as soon as you have one instruction. MOVE should be the easiest one, you just need to put input 1 (green line from image below) into the input of the multiplexer

then try making AND and OR or any other bitwise instruction you want to make. If possible, try to minimize the amount of chip used. e.g. use NAND implementation. But also, if other instruction also requires the use of AND or OR, then dont use NAND implementation.

If you're really fast, then try to do the ADD and 2's complement instruction. before you do the 2's complement, please text me!! My schematic for ADD can be found in our chat history, but if I were you, I would double check if my schematic is correct.

Finally, if you're very confident, you can do a shift instruction. But i reckon we should save this for the holiday

we have 8 instruction we can make.
  • 000: MOVE
  • 001: AND
  • 010: ORR
  • 011: your choice
  • 100: your choice
  • 101: ADD
  • 110: 2's complement
  • 111: Shift
